Testssl.sh WebUI

A convenient, secure, and web-based interface for testssl.sh, making TLS/SSL security checks easily accessible within your network.

Get Started Now View on GitHub

Key Features

Lightweight Go Gin Server

Utilizes a lightweight and efficient HTTP server, hosting a simple and intuitive web interface.

Docker-Native Deployment

Built directly on the official testssl.sh Docker image, no complex dependencies needed.

Private Network Scanning

Securely perform scans on internal domains without exposing your services to the public internet.

Real-time SSE Responses

Monitor scan progress and results instantly with Server-Sent Events (SSE) technology.

PDF Report Export

Export detailed scan results as a PDF report for easy sharing, archiving, or analysis.

Simplified Security Checks

Testssl-webui aims to make TLS/SSL security checks easily accessible for everyone.

How to Use

Get Testssl.sh WebUI up and running with a single Docker command.

Run in Background (Detached Mode)

This command runs the container in the background, freeing up your terminal.

docker run -d -it -p 8081:8081 vulcanshen2304/testssl-webui

Important Notes:

  • If you're using an ARM-based CPU (e.g., Apple M1/M2 Macs, Raspberry Pi, etc.), ensure you include the --platform=linux/amd64 flag. This tells Docker to perform cross-architecture emulation.
  • The -p 8081:8081 flag maps your host machine's port 8081 to the container's internal port 8081.

Once the container is running, open your web browser and navigate to:
http://localhost:8081
